043/**
044 * A byte buffer used by MINA applications.
045 * <p>
046 *   This is a replacement for {@link ByteBuffer}. Please refer to
047 *   {@link ByteBuffer} documentation for preliminary usage. MINA does not use NIO
048 *   {@link ByteBuffer} directly for two reasons:
049 *   <ul>
050 *     <li>It doesn't provide useful getters and putters such as <code>fill</code>,
051 *         <code>get/putString</code>, and <code>get/putAsciiInt()</code> enough.</li>
052 *     <li>It is difficult to write variable-length data due to its fixed capacity</li>
053 *   </ul>
054 * </p>
055 * 
056 * <h2>Allocation</h2>
057 * <p>
058 *   You can allocate a new heap buffer.
059 * 
060 *   <pre>
061 *     IoBuffer buf = IoBuffer.allocate(1024, false);
062 *   </pre>
063 * 
064 *   You can also allocate a new direct buffer:
065 * 
066 *   <pre>
067 *     IoBuffer buf = IoBuffer.allocate(1024, true);
068 *   </pre>
069 * 
070 *   or you can set the default buffer type.
071 * 
072 *   <pre>
073 *     // Allocate heap buffer by default.
074 *     IoBuffer.setUseDirectBuffer(false);
075 * 
076 *     // A new heap buffer is returned.
077 *     IoBuffer buf = IoBuffer.allocate(1024);
078 *   </pre>
079 * 
080 * </p>
081 * 
082 * <h2>Wrapping existing NIO buffers and arrays</h2>
083 * <p>
084 *   This class provides a few <tt>wrap(...)</tt> methods that wraps any NIO
085 *   buffers and byte arrays.
086 * </p>
087 * 
088 * <h2>AutoExpand</h2>
089 * <p>
090 *   Writing variable-length data using NIO <tt>ByteBuffers</tt> is not really
091 *   easy, and it is because its size is fixed at allocation. {@link IoBuffer} introduces
092 *   the <tt>autoExpand</tt> property. If <tt>autoExpand</tt> property is set to true, 
093 *   you never get a {@link BufferOverflowException} or
094 *   an {@link IndexOutOfBoundsException} (except when index is negative). It
095 *   automatically expands its capacity. For instance:
096 * 
097 *   <pre>
098 *     String greeting = messageBundle.getMessage(&quot;hello&quot;);
099 *     IoBuffer buf = IoBuffer.allocate(16);
100 *     // Turn on autoExpand (it is off by default)
101 *     buf.setAutoExpand(true);
102 *     buf.putString(greeting, utf8encoder);
103 *   </pre>
104 * 
105 *   The underlying {@link ByteBuffer} is reallocated by {@link IoBuffer} behind
106 *   the scene if the encoded data is larger than 16 bytes in the example above.
107 *   Its capacity will double, and its limit will increase to the last position
108 *   the string is written.
109 * </p>
110 * 
111 * <h2>AutoShrink</h2>
112 * <p>
113 *   You might also want to decrease the capacity of the buffer when most of the
114 *   allocated memory area is not being used. {@link IoBuffer} provides
115 *   <tt>autoShrink</tt> property to take care of this issue. If
116 *   <tt>autoShrink</tt> is turned on, {@link IoBuffer} halves the capacity of the
117 *   buffer when {@link #compact()} is invoked and only 1/4 or less of the current
118 *   capacity is being used.
119 * </p>
120 * <p>
121 *   You can also call the {@link #shrink()} method manually to shrink the capacity of the
122 *   buffer.
123 * </p>
124 * <p>
125 *   The underlying {@link ByteBuffer} is reallocated by the {@link IoBuffer} behind
126 *   the scene, and therefore {@link #buf()} will return a different
127 *   {@link ByteBuffer} instance once capacity changes. Please also note
128 *   that the {@link #compact()} method or the {@link #shrink()} method
129 *   will not decrease the capacity if the new capacity is less than the 
130 *   {@link #minimumCapacity()} of the buffer.
131 * </p>
132 * 
133 * <h2>Derived Buffers</h2>
134 * <p>
135 *   Derived buffers are the buffers which were created by the {@link #duplicate()},
136 *   {@link #slice()}, or {@link #asReadOnlyBuffer()} methods. They are useful especially
137 *   when you broadcast the same messages to multiple {@link IoSession}s. Please
138 *   note that the buffer derived from and its derived buffers are not 
139 *   auto-expandable nor auto-shrinkable. Trying to call
140 *   {@link #setAutoExpand(boolean)} or {@link #setAutoShrink(boolean)} with
141 *   <tt>true</tt> parameter will raise an {@link IllegalStateException}.
142 * </p>
143 * 
144 * <h2>Changing Buffer Allocation Policy</h2>
145 * <p>
146 *   The {@link IoBufferAllocator} interface lets you override the default buffer
147 *   management behavior. There are two allocators provided out-of-the-box:
148 *   <ul>
149 *     <li>{@link SimpleBufferAllocator} (default)</li>
150 *     <li>{@link CachedBufferAllocator}</li>
151 *   </ul>
152 *   You can implement your own allocator and use it by calling
153 *   {@link #setAllocator(IoBufferAllocator)}.
154 * </p>
155 * 
156 * @author <a href="http://mina.apache.org">Apache MINA Project</a>
157 */

276    /**
277     * Normalizes the specified capacity of the buffer to power of 2, which is
278     * often helpful for optimal memory usage and performance. If it is greater
279     * than or equal to {@link Integer#MAX_VALUE}, it returns
280     * {@link Integer#MAX_VALUE}. If it is zero, it returns zero.
281     * 
282     * @param requestedCapacity The IoBuffer capacity we want to be able to store
283     * @return The  power of 2 strictly superior to the requested capacity
284     */
285    protected static int normalizeCapacity(int requestedCapacity) {
286        if (requestedCapacity < 0) {
287            return Integer.MAX_VALUE;
288        }
289
290        int newCapacity = Integer.highestOneBit(requestedCapacity);
291        newCapacity <<= (newCapacity < requestedCapacity ? 1 : 0);
292        
293        return newCapacity < 0 ? Integer.MAX_VALUE : newCapacity;
294    }

385     *  |XXXXXXXX|          |
386     *  +--------+----------+
387     *   ^       ^          ^
388     *   |       |          |

390     *

400     *

406     * @return the underlying NIO {@link ByteBuffer} instance.

437     * Changes the capacity and limit of this buffer so this buffer get the

447     *

449     *  +--------+----------+
450     *  |XXXXXXXX|          |
451     *  +--------+----------+
452     *   ^       ^          ^
453     *   |       |          |
454     *  pos    limit     capacity
455     *  
456     * ( pos + V )  &lt;= L, no change :
457     * 
458     *   0       L          C
459     *  +--------+----------+
460     *  |XXXXXXXX|          |
461     *  +--------+----------+
462     *   ^       ^          ^
463     *   |       |          |
464     *  pos    limit   newCapacity
465     *  
466     * You can still put ( L - pos ) bytes in the buffer
467     *  
468     * ( pos + V ) &gt; L & ( pos + V ) &lt;= C :
469     * 
470     *  0        L          C
471     *  +------------+------+
472     *  |XXXXXXXX:...|      |
473     *  +------------+------+
474     *   ^           ^      ^
475     *   |           |      |
476     *  pos       newlimit  newCapacity
477     *

479     *  
480     *

482     * 
483     *   0       L          C
484     *  +-------------------+----+
485     *  |XXXXXXXX:..........:....|
486     *  +------------------------+
487     *   ^                       ^
488     *   |                       |

490     *                           |

576    /**
577     * Changes the capacity of this buffer so this buffer occupies as less
578     * memory as possible while retaining the position, limit and the buffer
579     * content between the position and limit. 
580     * <br>
581     * <b>The capacity of the buffer never becomes less than {@link #minimumCapacity()}</b>
582     * <br>. 
583     * The mark is discarded once the capacity changes.
584     * <br>

586     * as possible, dividing by two the initial capacity until it can't without
587     * obtaining a new capacity lower than the {@link #minimumCapacity()}. For instance, if 
588     * the limit is 7 and the capacity is 36, with a minimum capacity of 8, 
589     * shrinking the buffer will left a capacity of 9 (we go down from 36 to 18, then from 18 to 9).  
590     * 
591     * <pre>
592     *  Initial buffer :
593     *   
594     *  +--------+----------+
595     *  |XXXXXXXX|          |
596     *  +--------+----------+
597     *      ^    ^  ^       ^
598     *      |    |  |       |
599     *     pos   |  |    capacity
600     *           |  |
601     *           |  +-- minimumCapacity
602     *           |
603     *           +-- limit
604     * 
605     * Resulting buffer :
606     * 
607     *  +--------+--+-+
608     *  |XXXXXXXX|  | |
609     *  +--------+--+-+
610     *      ^    ^  ^ ^
611     *      |    |  | |
612     *      |    |  | +-- new capacity
613     *      |    |  |
614     *     pos   |  +-- minimum capacity
615     *           |
616     *           +-- limit
617     * </pre>
618     *           
619     * @return The modified IoBuffer instance
620     */
